> 검토 기준: 15~20분 투자 발표, 대상은 투자사 대표

## 2. 문제점

### A. 발표 흐름 관련

**1) 시연 4회가 흐름을 과도하게 끊는다**
- 시연은 "말로 설명하기 어려운 것을 보여주는" 용도인데, 4번은 너무 많다.
- 특히 시연 3(마음숲 도감)은 캐릭터 슬라이드 바로 뒤에 나와서 "교육 근거 → 사업성"으로 넘어가는 전환을 방해한다.
- 15~20분에 시연 4회 = 시연당 2~3분 가정 시 시연만 8~12분. 설명 시간이 부족해진다.

**2) 슬라이드 5(해결책) → 6(시나리오) → 7(시연1) 3장 연속으로 같은 것을 반복**
- 슬라이드 5에서 "스크린 타임을 프라임 타임으로" 개념 설명
- 슬라이드 6에서 서연이 시나리오로 같은 내용을 반복
- 시연 1에서 또 온보딩을 직접 보여줌
- 투자자 입장에서 "알겠는데, 다음은?" 이 될 수 있음

**3) "돈이 되겠다"는 확신이 너무 늦게 온다**
- 비즈니스 모델이 슬라이드 15(19장 중 15번째)에 나온다.
- 투자자가 10분 넘게 "좋은 제품이네" 상태에서 "근데 돈이 될까?"를 품고 있게 됨.
- 슬라이드 9(왜 돈을 내는가)에서 단서가 제공되지만, 구체적 재무 모델/숫자가 없어서 확신까지 가지 못함.

**4) 시장 → 경쟁 분석 순서의 어색함**
- 현재: 시장(3) → 경쟁 한계(4) → 해결책(5)
- 이 순서 자체는 나쁘지 않으나, 슬라이드 4의 발표자 노트가 3분 이상 걸릴 분량. 시장+경쟁을 합쳐 2분 이내로 끝내고 빠르게 솔루션으로 넘어가야 청중이 이탈하지 않음.

**5) 경쟁 우위(슬라이드 16)와 경쟁 분석(슬라이드 4)이 12장 떨어져 있다**
- 4번에서 "빈자리가 있다"고 했는데, "왜 우리가 그 자리를 채울 수 있는가"가 16번에서야 나온다.
- 그 사이 12장 동안 "정말 너희가 할 수 있어?"라는 의문이 방치됨.

### B. 각 슬라이드 역할 관련

**1) 중복/합칠 수 있는 슬라이드**
- 슬라이드 5(해결책)과 슬라이드 6(시나리오): 시나리오가 해결책을 설명하는 방식이므로 하나로 합칠 수 있음. 또는 시나리오를 발표자 노트로 구술하고 화면은 5번만 유지.
- 슬라이드 10(비즈니스 모델)과 슬라이드 9(왜 돈을 내는가): 내용이 연속적. 하나의 "수익 구조" 슬라이드로 합치거나, 최소한 연달아 배치해야 함 (현재도 거의 연속이지만 시연 4가 사이에 끼어있음).

**2) 한 슬라이드에 과부하**
- 슬라이드 4(기존 해결책 한계): 발표자 노트가 매우 길다. Khan Kids, Lingokids, 뽀로로/핑크퐁까지 다루는데 이것을 1분 이내로 전달하기 어려움.
- 슬라이드 10(비즈니스 모델): 무료/프리미엄 비교 + 해지 방어 4가지 + 4단계 확장 로드맵이 한 장에. 너무 많음.

**3) 약한 슬라이드**
- 슬라이드 14(IP 확장)와 슬라이드 10(비즈니스 모델)의 Phase 2~4가 거의 동일한 내용. IP 확장 슬라이드가 비즈니스 모델의 반복처럼 느껴짐.

### C. 투자자 관점

**1) 첫 3장 안에 "이 사업이 뭔지" 이해 가능한가?**
- 슬라이드 1: 카피 (이해됨)
- 슬라이드 2: 문제 (공감됨)
- 슬라이드 3: 시장 규모 (돈 냄새)
- 결론: 3장까지는 "유아 에듀테크, 부모 참여형"으로 이해 가능. **합격.**

**2) "돈이 되겠다"는 확신 시점**
- 현재 구조에서는 슬라이드 13(왜 돈을 내는가) + 15(비즈니스 모델) 시점에서야 형성.
- 이때쯤이면 발표 시작 후 12~14분. 너무 늦다.
- **재무 예측(TAM/SAM/SOM, 3년 매출 시나리오)이 완전히 빠져있다.** 투자자가 가장 궁금해할 숫자가 없음.

**3) 팀 소개가 없다**
- 투자 발표에서 "왜 이 팀이 할 수 있는가"는 필수.
- 슬라이드 16(경쟁 우위)에서 "학원에서 시작, 글로벌 캐릭터 사업 경험"을 언급하지만, 구체적 팀 멤버/경력이 없으면 신뢰가 불완전.

**4) 투자 요청 금액이 없다**
- Ask(투자 요청 금액, 밸류에이션, 자금 용도)가 완전히 빠져있다.
- 투자 발표의 마지막은 "얼마 투자해주시면, 이것을 하겠다"로 끝나야 한다.

### D. 빠진 내용

| 항목 | 현재 상태 | 필요도 |
|------|-----------|--------|
| AI 기능 전략 | 완전히 빠짐 | 중 — 에듀테크에서 AI 활용은 투자자 관심사 |
| 재무 모델/숫자 | 완전히 빠짐 | 상 — 투자 발표 필수 |
| 팀 소개 | 빠짐 | 상 — 초기 투자에서 팀이 90% |
| 투자 요청(Ask) | 빠짐 | 상 — 발표의 목적 자체 |
| 트랙션/검증 데이터 | 빠짐 | 중 — 있으면 넣어야 함 (없으면 검증 계획이라도) |
| 시장 진입 전략(GTM) | 빠짐 | 중 — 어떻게 첫 1000명을 모을 것인가 |

## 3. 개선 제안

### A. 구조 재배치 (20장 → 18~20장)

**제안 순서:**

| # | 슬라이드 | 변경 사항 |
|---|----------|-----------|
| 1 | 표지 | 유지 |
| 2 | 문제 | 유지 |
| 3 | 시장 규모 | 유지 |
| 4 | 해결책 + 시나리오 | 5+6 통합. 화면은 Before/After, 구술로 서연이 이야기 |
| 5 | **시연 1**: 온보딩 | 유지. "직접 보시죠" 한번만 |
| 6 | 사용자 경험 흐름 | 유지 |
| 7 | **시연 2**: Blackout + Handoff | 핵심 차별점이므로 유지 |
| 8 | 캐릭터 + 커리큘럼 | 7+8 통합. 캐릭터 비주얼 보여주면서 CASEL 매핑 한 장에 |
| 9 | **시연 3**: 마음숲 도감 + 부모 대시보드 | 시연 3,4 통합. 도감 잠깐 → PIN → 부모 화면까지 하나의 흐름으로 |
| 10 | 왜 돈을 내는가 + 비즈니스 모델 | 9+10 통합 또는 연속 배치 (시연 끼지 않도록) |
| 11 | 경쟁 우위 (왜 우리만) | 앞으로 이동. 해결책 설명 직후에 "왜 우리"를 바로 답변 |
| 12 | 재무 모델 (신규) | TAM/SAM/SOM, 3년 매출 시나리오, 주요 가정 |
| 13 | 팀 소개 (신규) | 핵심 멤버 2~3인, 관련 경력, 자문위원 |
| 14 | 로드맵 | 유지 |
| 15 | IP 확장 | 유지 (단, 비즈니스 모델과 중복 줄이기) |
| 16 | 리스크와 대응 | 유지 |
| 17 | 투자 요청 Ask (신규) | 요청 금액, 밸류에이션, 자금 용도 breakdown |
| 18 | 마무리 | 유지 |

### B. 시연 최적화

**현재 4회 → 3회로 축소 권장**

- 시연 1 (온보딩): 유지. 첫인상이 중요.
- 시연 2 (Blackout/Handoff): 유지. 핵심 차별점을 눈으로 봐야 함.
- 시연 3+4 통합: 마음숲 도감을 10초 보여준 후, PIN 입력하여 부모 대시보드까지 하나의 연속 흐름으로 시연. 별도 슬라이드 분리 불필요.

**시연 배치 원칙:**
- 시연 직전 슬라이드에서 "왜 이걸 봐야 하는지" 기대를 만들 것
- 시연 직후 슬라이드에서 "방금 본 것의 사업적 의미"를 연결할 것
- 시연 중간에 멈추지 말고 30~60초 내로 완료할 것

### C. 필수 추가 슬라이드

**1) 재무 모델 (1장)**
- 3년 매출 시나리오 (보수/기본/낙관)
- 주요 가정: DAU, 전환율, ARPU, 해지율
- BEP 시점
- 단위 경제학: CAC vs LTV

**2) 팀 소개 (1장)**
- 대표: 교육 사업 경력, 성과
- CTO/개발: 기술 역량
- 캐릭터/디자인: IP 사업 경험
- 자문위원: 아동발달 전문가

**3) 투자 요청 Ask (1장)**
- 요청 금액
- Pre-money 밸류에이션
- 자금 용도: 콘텐츠 제작 X%, 개발 X%, 마케팅 X%
- 마일스톤: "이 돈으로 N개월 내에 이것을 달성하겠다"

### D. 발표자 노트 분량 조정

**현재 문제:** 슬라이드 4(기존 해결책)의 발표자 노트가 약 400자. 2분 이상 소요. 15~20분 발표에서 한 슬라이드에 2분은 과함.

**제안:**
- 슬라이드 2~4는 각 45초~1분 이내로 압축
- "5분 안에 문제와 기회를 설명하고, 5분 안에 제품을 보여주고, 5분 안에 사업성과 팀을 증명한다" 구조
- Khan Kids, Lingokids 비교는 표 하나로 충분. 구술로 장황하게 설명하지 말 것

### E. AI 기능 전략 추가 방법

- 별도 슬라이드를 만들 필요 없음
- 로드맵 또는 비즈니스 모델 슬라이드에 한 줄 추가: "Phase 2에서 AI 기반 맞춤 커리큘럼 추천, 부모 대화 가이드 개인화 도입"
- 발표자 노트에서 질문 대비용으로 준비: "AI는 어떻게 쓸 건가요?" → "아이별 감정 학습 이력 기반으로 다음 에피소드를 추천하고, 부모 가이드 대사를 아이 이름과 상황에 맞게 개인화합니다."

## 4. 요약: 핵심 액션 3가지

1. **시연 4회 → 3회 축소.** 시연 3+4를 하나로 합쳐 흐름 끊김 최소화.
2. **재무 모델 + 팀 소개 + Ask 3장 추가.** 투자자가 결정에 필요한 정보가 현재 완전히 빠져있음. 이것 없이는 "좋은 발표"는 되지만 "투자 받는 발표"는 되지 않음.
3. **"돈이 되겠다" 시점을 앞당기기.** 비즈니스 모델/재무를 슬라이드 10~12 위치로 이동. 발표 절반 시점에서 투자자가 "이건 될 수 있겠다"를 느끼게 할 것.
